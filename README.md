<p>My Certificates is an Android application that allows users to view details of X509 security certificates.  </p>

<h2><a id="user-content-google-play-store-link" class="anchor" href="#google-play-store-link" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Google Play Store Link</h2>

<a href="https://play.google.com/store/apps/details?id=com.wesbunton.projects.mycertificates">https://play.google.com/store/apps/details?id=com.wesbunton.projects.mycertificates</a></p>

<h2><a id="user-content-getting-started" class="anchor" href="#getting-started" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Getting Started</h2>

<p>These instructions will get you a copy of the project up and running on your local machine for development and testing purposes. You can use the latest version from the Google Play Store now! If you'd like to build it from source, you can do so from the master branch of this repo.</p>

<h2><a id="user-content-quickstart-guide" class="anchor" href="#quickstart-guide" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Quickstart Guide</h2>

<p>Using the My Certificate Android application is simple. There are two buttons in the interface. One lists the available user certificates from KeyChain, another from a local file. Allowing access to one of the KeyChain certificates will allow the application to parse the certificate and display the information to the user. Similarly, supplying the password for a P12/PFX file will allow parsing of it.</p>

<p>Note that none of the certificate or key information is stored within the application. Once a user goes back to the main screen, all of the certificate information is forgotten. In fact, to display the information again, the user will need to again perform the explicit allowing of access to the certificate.</p>

<h3><a id="user-content-note-about-lock-screen" class="anchor" href="#note-about-lock-screen" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Note about lock screen:</h3>

<p>Because the Android platform has security in mind when implementing the KeyChain security certificate store, the platform does require that users have a lock screen enabled to successfully store an X509 user certificate. You can store CA certificates for doing things like HTTPS/TLS connections without a lock screen.</p>

<p>When the application is launched for the first time, there is a tip shown reminding users that they system might prompt them to enable a lock screen during app use to protect their user certificates.</p>

<h2><a id="user-content-use-cases" class="anchor" href="#use-cases" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Use cases:</h2>

<p><strong>1. Install new certificate</strong><br>
Prerequisite:<br>
You need to have an X509 certificate within a P12 file local to your Android device.  </p>

<p>Steps:<br>
1. Launch My Certificates app.<br>
2. Select the "Select Certificate from Store" button. You'll see the Android user certificate selection screen. At the bottom of this dialog is an <strong>Install</strong> button. Select this.
3. Browse to your new certificate file. Select it.<br>
4. You'll be prompted to enter a password if the file is password protected (recommended for security certificates with private keys).<br>
5. You'll be prompted to enter a name for the certificate. This is how you'll see the certificate labeled within the list of user certificates.<br>
6. Finally, select whether you plan to use this certificate for VPN/app use or only for Wi-Fi authentication. Then tap <strong>OK</strong>.  </p>

<p><strong>2. View Details of installed certificate</strong><br>
Prerequisite:<br>
You need to have a user certificate installed on the device. Note that this is different from simply having a CA certificate.</p>

<p>Steps:<br>
1. Launch My Certificates app.<br>
2. Select the "Select Certificate" button. You'll see the Android user certificate selection screen. Select the certificate to examine, and tap the <strong>ALLOW</strong> button.<br>
3. You'll be shown a list of details for that specific user certificate. If there's other certificates within a certificate chain that include this certificate, such as its issuing CA/RA certificate, you can swipe sideways to view the details of those certificates as well.  </p>

<p><strong>3. View Details of certificate from local file</strong><br>
Prerequisite:<br>
File containing a certificate is present on the device.</p>

<p>Steps:<br>
1. Launch My Certificates app.<br>
2. Select the "Select Certificate from File" button. You'll see the Android file selector prompt. Select a certificate file containing a P12/PFX or PEM format certificate.<br>
3. If you've selected a P12/PFX, you'll be prompted to enter the password to decrypt it.<br>
3. You'll be shown a list of details for that specific user certificate.  </p>

<h3><a id="user-content-new-features" class="anchor" href="#new-features" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>New Features!</h3>

<p>You can now select X509 certificates that are simply stored in PEM, P12, or PFX format in a local file. With this implementation, the user can inspect certificate files without having to install them, which is inconvenient due to Android clear-all certificate implementation.</p>

<p>Feel free to submit an issue to add a feature request!</p>
